# 物品多图与重新解析实施计划

1. 迁移资产照片字段并更新移动端领域类型。
   - 验证：旧路径转换为单元素数组，数据库拒绝 0 张和超过 5 张。
2. 升级解析接口为 1–5 个图片 URL，并补服务端测试。
   - 验证：全部图片进入 OpenAI 请求，非法数量返回校验错误。
3. 接入系统相机与相册多选，建立可复用的照片选择与封面排序界面。
   - 验证：最多 5 张、至少 1 张，删除与设封面行为有最小单元检查。
4. 改造新建流程为“选图 → 多图解析 → 确认 → 保存”。
   - 验证：成功保存全部路径；上传或解析失败会清理本次文件。
5. 扩展编辑页，支持重拍、添加、删除、设封面与可选重新解析。
   - 验证：解析结果先预览，关闭解析可直接保存，失败不覆盖表单或旧照片。
6. 更新列表和详情页多图展示，并接回自动估价。
   - 验证：列表使用封面，详情可浏览全部照片，估价失败仍保留更新。
7. 运行定向测试和移动端构建检查。
   - 验证：服务端测试、移动端 lint/TypeScript、Android/iOS bundle 通过。
