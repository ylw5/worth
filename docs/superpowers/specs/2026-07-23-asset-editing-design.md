# 物品信息编辑设计

## 目标

物品详情页支持编辑已有物品信息。保存后自动重新估价；如果估价失败，仍保留已保存的信息，并允许用户稍后在详情页手动刷新价格。

## 交互

- 详情页右上角显示“编辑”按钮。
- 点击后进入独立编辑页，表单预填当前物品信息。
- 可编辑名称、品牌、型号、规格、分类、成色和估价搜索词。
- 照片、当前价格、价格历史和添加时间保持只读。
- 名称或估价搜索词为空时阻止保存并显示现有校验文案。
- 保存期间禁用提交按钮。
- 保存并重新估价成功后返回详情页，详情和列表展示最新数据。
- 信息保存成功但估价失败时保留修改，在编辑页提示“信息已保存，估价失败，可稍后刷新价格”，用户返回详情页后可手动刷新。
- 信息保存失败时留在编辑页并显示错误。

## 实现

- 从现有确认页提取共享物品字段表单，创建页和编辑页复用同一字段布局与分类选择。
- 在资产数据模块新增一个按资产 ID 更新 `AssetInput` 字段的方法，继续依赖现有 Supabase RLS 限制所有者权限。
- 新增编辑路由；加载现有资产后初始化表单。
- 保存顺序固定为：更新资产 → 请求估价 → 记录估价 → 刷新资产详情、估价历史和资产列表缓存 → 返回详情页。
- 更新成功后不回滚信息；估价或记录估价失败走已确认的降级提示。
- 不新增数据库迁移、服务端接口或依赖。

## 验证

- 静态检查通过。
- 更新方法只提交允许编辑的 `AssetInput` 字段。
- 编辑页能预填、校验、保存并刷新相关查询。
- 模拟估价失败时，已保存的信息不丢失且显示可恢复提示。
- 创建确认页继续使用共享表单，原有创建流程不变。
