# 物品买入信息设计

## 目标

用户在添加或编辑物品时，可以选填实际买入日期和买入价格，并在物品详情页查看这两项信息。

## 数据

- `assets` 表新增可空的 `purchase_date date`。
- `assets` 表新增可空的 `purchase_price numeric(12, 2)`，存在时必须大于 0。
- `AssetInput` 和 `Asset` 增加对应字段；AI 照片识别返回的初始值均为空。
- 买入信息只用于资产记录，不参与市场估价。

## 交互

- 共享物品表单增加“实际买入日期”和“实际买入价格”，因此添加页和编辑页保持一致。
- 日期使用现有文本输入框，格式为 `YYYY-MM-DD`。
- 价格使用现有文本输入框并启用数字键盘。
- 两项都允许留空，表示未知。
- 物品详情页在基础信息区展示已填写的买入日期和买入价格；未填写时显示“—”。
- 资产列表卡片保持不变。

## 校验与错误

- 日期为空时通过；非空时必须是实际存在的 `YYYY-MM-DD` 日期。
- 价格为空时通过；非空时必须是大于 0 的数字。
- 表单校验失败时保留当前输入并显示明确提示，不发送保存请求。
- 数据库价格约束阻止绕过客户端写入非正价格。

## 实现范围

- 新增一条 Supabase migration。
- 扩展移动端资产类型、共享表单、创建初始值、编辑初始值与详情展示。
- 复用现有组件和格式化方式，不新增依赖、日期选择器、状态管理或服务端接口。

## 验证

- 最小测试覆盖有效日期、无效日期、有效价格、零或负数价格以及两个字段留空。
- 运行移动端 lint 和 TypeScript 检查。
- 检查创建与编辑保存的数据包含两个新字段，详情页能正确展示。
