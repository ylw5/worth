# 添加物品先拍照设计

## 目标

点击“我的资产”右上角加号后，直接打开系统全屏相机。用户确认第一张照片后进入现有录入页；取消则返回“我的资产”。

## 交互

- 首页加号进入带相机意图的录入路由。
- 录入页识别相机意图后请求相机权限，并立即打开 `expo-image-picker` 的系统相机。
- 用户取消拍摄时返回“我的资产”，不保留照片或创建草稿。
- 用户确认照片时只接收第一张，将其加入现有上传、识别和抠图流程，再显示现有录入表单。
- 录入页内原有继续拍照和从相册选择的能力保持不变。
- 系统相机的按钮外观由 iOS 或 Android 决定。

## 实现范围

- 复用已安装的 `expo-image-picker`，不新增相机依赖。
- 复用现有照片转换和处理流程，不引入全局临时状态。
- 仅首页右上角加号启用先拍照流程；其他现有录入入口保持原行为。
- 相机权限被拒绝或相机启动失败时提示原因，随后返回“我的资产”。

## 验证

- 点击首页加号会直接打开系统全屏相机。
- 取消后回到“我的资产”，没有新增照片或草稿。
- 确认后录入页包含第一张照片，并自动开始现有处理流程。
- 权限拒绝和相机异常不会停留在空白录入页。
- TypeScript、lint 和现有测试通过。
