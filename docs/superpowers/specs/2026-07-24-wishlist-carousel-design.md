# 心愿单横向 Carousel 设计

## 目标

将心愿单首页的纵向卡片列表改为横向 carousel：主卡居中，左右露出相邻卡边缘，左右滑切换；删除与进入详情交互保持不变。

## 布局

- 列表改为横向 `FlatList`。
- 卡宽为屏宽的约 86%，卡片间距固定；通过 `snapToInterval` 与 `decelerationRate="fast"` 吸附到卡位。
- 首尾用水平 `contentContainer` padding，使第一张与最后一张居中，左右露出邻卡约 8–12%。
- 多张心愿时，carousel 下方显示圆点指示器；仅一张时不显示圆点。
- 空态、加载、错误仍纵向居中展示，不进入 carousel。
- 页面不再纵向滚动卡片列表；卡内备注过长时截断（`numberOfLines`），避免撑破卡片高度。

## 交互

- 左右滑切换当前心愿。
- 删除：确认框与删除逻辑不变。
- 进入详情：现有路由跳转不变。
- 仅一张时不强制滑动手势行为变化（列表仍可渲染，但无可切换邻卡）。

## 实现范围

- 主要修改 `mobile/src/app/(tabs)/(wishlist)/index.tsx`；若卡片 JSX 过长可抽小组件，不强制。
- 不新增依赖（使用已有 React Native `FlatList`）。
- 不改数据层、不改添加页、不改进度计算逻辑。

## 验证

- 多张心愿时可左右滑，主卡居中且邻卡边缘可见；圆点与当前索引一致。
- 一张时无圆点，卡片正常展示。
- 空态、加载、错误展示正常。
- 删除与进入详情仍可用。
