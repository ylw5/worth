# 心愿单设计

## 目标

新增“心愿单”底部 Tab。用户可以查看、添加和删除自己的心愿物品。

## 数据

每条心愿物品包含：

- 名称：必填，去除首尾空格后不能为空。
- 目标价格：必填，必须是大于 0 的有效数字。
- 备注：可选。

新增 `wishlist_items` 表，记录所属用户和创建时间。通过 Supabase RLS 只允许登录用户读取、添加和删除自己的记录，列表按创建时间倒序展示。

## 交互

- 底部导航在“资产”和“账号”之间新增“心愿单”。
- 心愿单首页显示名称、格式化后的目标价格和可选备注。
- 空列表显示添加引导。
- 标题栏右上角显示添加按钮，点击进入独立添加页。
- 保存期间禁用提交按钮；保存成功后刷新列表并返回心愿单。
- 每条记录提供删除按钮，点击后使用系统确认框二次确认。
- 删除成功后立即刷新列表。

## 错误处理

- 表单校验失败时停留在添加页并显示明确提示。
- 保存失败时保留已填写内容并显示错误。
- 删除失败时保留该记录并显示错误。

## 实现范围

- 新增心愿单 Tab、列表页和添加页。
- 新增最小的心愿物品类型和 Supabase 查询、创建、删除方法。
- 新增数据库迁移，包括表、索引和 RLS 策略。
- 复用现有颜色、货币格式化、加载状态和错误状态。
- 不新增依赖，不复用资产照片识别流程。
- 不支持链接、图片、编辑、排序、购买状态或 AI。

## 验证

- 静态检查通过。
- 名称为空、目标价格为空、非数字或不大于 0 时不能保存。
- 登录用户只能查看和删除自己的心愿物品。
- 新增成功后列表出现记录，确认删除后记录消失。
