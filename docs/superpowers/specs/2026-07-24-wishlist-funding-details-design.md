# 心愿资金明细设计

## 目标

在心愿卡片下方增加资金明细模块，分别展示用户通过“忍住消费”和“已卖闲置”积累的记录与金额，并让心愿主进度使用两类金额的合计。

## 数据口径

- “忍住消费”读取当前用户 `spending_resolutions.confirmed_at` 非空的记录，金额使用 `amount`。
- “已卖闲置”读取当前用户 `asset_sales` 记录，金额使用实际 `sale_price`。
- 心愿主进度金额为两类金额之和，百分比继续使用合计金额除以当前心愿的 `target_price`，进度条最大显示 100%。
- 两类金额均为用户级累计金额，所有心愿卡共享，不分配或绑定到某一个心愿。
- 金额由客户端基于当前用户通过 RLS 可见的记录计算，不新增数据库迁移、RPC 或缓存字段。

## 界面

心愿卡片 Carousel 与圆点指示器下方增加一个白色明细模块，复用现有颜色、圆角、间距和排版。

模块包含两个可切换的 Tab：

1. `忍住消费`
2. `已卖闲置`

当前 Tab 显示选中状态，并在内容区顶部展示该类记录的总金额。明细按时间倒序排列：

- “忍住消费”每行显示商品名称、确认省下金额和确认时间。
- “已卖闲置”每行显示物品名称、实际成交价和成交日期。

没有记录时显示对应的简短空状态。Tab 和明细行使用原生 React Native 组件，不新增依赖；本期不增加搜索、筛选、动画或明细跳转。

## 数据流与状态

心愿页并行读取：

- 心愿列表；
- 已确认的忍住消费记录；
- 已卖闲置成交记录及对应物品名称。

三类数据加载完成后计算两个分类总额和合计金额。合计金额用于每张心愿卡的主金额与进度，分类总额和记录用于下方 Tab。

- 任一资金明细查询仍在加载时，不显示不完整的心愿进度和明细模块。
- 任一资金明细查询失败时，复用现有错误状态，不把失败数据当作零。
- 页面重新获得焦点时刷新两类资金明细，使新确认的省下记录和新成交记录能够更新。

## 实现范围

- 扩展现有消费决策查询，使其返回展示明细需要的已确认记录。
- 新增一次读取成交记录和物品名称的查询，避免复用完整资产列表或逐件查询成交记录。
- 在心愿页复用现有金额格式化、React Query、颜色和排版配置。
- 扩展现有纯函数测试，覆盖两类金额分别求和及合计进度。

## 验证

- 只累计已确认的忍住消费记录。
- 两类金额分别求和正确，心愿主进度使用两者合计。
- 两个 Tab 的总金额与列表内容匹配。
- 明细按时间倒序显示。
- 任一分类无记录时显示正确空状态。
- 页面重新获得焦点后两类明细均会刷新。
- 运行相关 Node 测试、TypeScript 检查和代码格式检查。

## 不在本期

- 将资金记录分配或绑定到具体心愿。
- 使用预估市价代替实际成交价。
- 新增数据库聚合接口、物化汇总或用户总额缓存。
- 明细搜索、筛选、分页、动画或点击跳转。
