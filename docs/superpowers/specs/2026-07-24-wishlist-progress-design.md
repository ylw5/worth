# 心愿进度卡片设计

## 目标

将心愿单列表中的每个心愿改为进度卡片，展示心愿标题、用户经 AI 评估后确认不买所省下的总金额、心愿目标金额和完成百分比。

## 数据口径

- 目标金额使用当前心愿的 `wishlist_items.target_price`。
- 已省金额为当前用户所有 `spending_resolutions.confirmed_at` 非空记录的 `amount` 总和。
- 已省金额是用户级总额，所有心愿卡片共享，不分配到具体心愿。
- 百分比为 `已省金额 / 目标金额`，显示四舍五入后的整数，进度条最大为 100%。

客户端直接读取当前用户可见的已确认记录并求和，复用现有 RLS，不新增数据库迁移、RPC 或缓存字段。

## 界面

每张卡片按以下顺序展示：

1. 顶部显示心愿标题，右侧保留删除操作。
2. 主数字显示 `¥已省金额 / 目标金额`。
3. 下方显示蓝色进度条和整数百分比。
4. 有备注时继续显示备注。
5. 保留“查看今日卖出方案”入口。

卡片复用现有颜色、圆角、间距和货币格式化风格，不新增组件库。进度条使用 React Native `View` 实现；目标达成或超额时宽度封顶为 100%。

## 数据流与状态

心愿单页面并行查询心愿列表和已确认的忍住消费记录。消费记录查询成功后在客户端求和，并将同一个总额用于每张心愿卡片。

- 心愿列表加载和错误状态维持现状。
- 总额加载期间不显示不完整的进度卡片。
- 总额读取失败时复用现有错误状态，不把错误数据当作零。
- 没有已确认记录时，总额和进度均为零。

## 验证

- 只累计 `confirmed_at` 非空的记录。
- 多条已确认记录正确求和。
- 零记录时总额和百分比为零。
- 百分比按整数显示，进度条在超过目标时封顶为 100%。
- 运行移动端最小测试和 TypeScript 检查。

## 不在本期

- 将省下金额分配或绑定到具体心愿。
- 将卖出闲置所得计入本卡片进度。
- 新增数据库聚合接口、用户总额缓存、动画或成就反馈。
