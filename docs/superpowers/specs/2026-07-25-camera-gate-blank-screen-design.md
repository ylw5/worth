# 相机先拍流程空白门闩设计

## 目标

消除「我的资产」加号进入系统相机、以及取消拍照返回首页时，录入表单一闪而过的问题。等待相机期间只显示同色空白页。

本设计是对 `2026-07-24-camera-first-add-asset-design.md` 的补丁：交互意图不变，只补齐相机前后的视觉门闩。

## 问题

`/capture?camera=1` 挂载后会立刻渲染完整录入表单，再由 `useEffect` 打开系统相机。因此：

- 进入时：表单先闪一下，再切到相机。
- 取消时：相机关闭后表单再次露出，随后才 `replace` 回「我的资产」。

## 交互

- `camera=1` 且尚未确认首张照片时：只显示应用背景色空白页，隐藏录入 header 与表单内容。
- 用户确认首张照片后：解除门闩，显示现有录入页（照片区 + 表单 + 保存），并走现有上传/识别/抠图流程。
- 用户取消拍摄、权限拒绝、或相机启动失败：保持空白，再按现有逻辑返回「我的资产」（失败仍弹出「无法拍照」提示）。
- 无 `camera=1` 的录入入口行为不变，直接显示空表单。

## 实现范围

- 仅改 `mobile/src/app/(tabs)/(capture)/index.tsx`（必要时微调该页 `Stack.Screen` options）。
- 不改首页路由、不改 `AssetPhotoPicker`、不新增全局状态或相机依赖。
- 继续复用 `expo-image-picker` 与现有 `addPhotos` 流程。

## 门闩规则

- 开启：路由带 `camera=1`，且首张照片尚未确认进入处理流程。
- 关闭：首张照片确认并交给现有 `addPhotos` 之后。
- 门闩期间 UI：纯背景色 `View`，`headerShown: false`；不渲染 `AssetPhotoPicker`、`AssetFormFields`、错误文案与保存按钮。

## 验证

- 点首页加号：看不到录入表单，直接进入系统相机。
- 取消拍照：回「我的资产」过程中不出现表单闪现。
- 确认拍照：进入带首张照片的录入页，并开始现有处理流程。
- 权限拒绝 / 相机失败：提示后回首页，不停留在空白录入页展示表单。
- 空态「添加」等非相机入口仍直接打开录入表单。
