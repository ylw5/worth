# Worth 每日市场估值与详情曲线设计

## 目标

为未售物品每日采集一次二手市场参考估值，并在资产详情页展示历史估值曲线。估值来自闲鱼在售样本，不描述为真实成交价。

## 刷新范围

- 只刷新状态为“使用中”“闲置”“出售中”的物品。
- 已售物品停止创建新的估值任务，已有历史曲线继续保留。
- 每天按 `(user_id, market_key, run_date)` 最多执行一个任务。
- `market_key` 沿用现有数据库生成规则；同一用户下搜索词和成色相同的物品共享一次采集结果。

## 后台流程

1. Cloudflare Cron 每日触发一次调度。
2. 调度调用现有 `enqueue_daily_market_runs()`，为需要更新的行情创建任务。
3. 每个任务启动一个 Workflow，领取任务后搜索闲鱼在售商品并按商品 ID 去重。
4. 沿用现有 AI 筛选规则排除配件、错误型号和关键规格不符的候选。
5. 至少有 5 个有效样本时计算中位数及价格区间，并调用 `complete_market_run()`。
6. 数据库把结果写入所有共享该 `market_key` 的未售物品当日快照，同时更新物品的最新估值。

现有 `(asset_id, snapshot_date)` 唯一约束保证 Workflow 重试只更新当日快照，不生成重复曲线点。

## 失败处理

- 搜索、筛选或写入失败时将任务标记为失败，不覆盖最后一次成功估值。
- 样本少于 5 个时不发布新快照。
- 详情页继续展示最近成功数据，并提示“本次更新失败，仍展示上次结果”。
- 密钥只保存在 Cloudflare Secrets；日志不记录闲鱼 Cookie、Supabase Service Role Key 或模型凭据。

## 详情页

当前参考市价卡继续作为金额主视觉，展示最新中位数、参考区间、样本数、数据源和更新时间。

其后增加“市场趋势”卡：

- 支持 30 天、90 天、全部三个范围。
- 按日期升序绘制每日估值中位数折线和数据点。
- 展示所选范围的涨跌额、涨跌比例、最高值和最低值。
- 没有快照时显示“暂无行情”。
- 只有一个快照时显示单点和“行情积累中”。
- 保留页面底部现有估值历史列表，作为可核对记录。

第一版使用 React Native 原生 `View` 绘制静态折线，不新增图表依赖，也不提供拖拽、缩放或悬浮提示。

## 数据读取

资产详情在资产加载成功后并行读取：

- 该物品的 `market_snapshots`，按日期升序；
- 该 `market_key` 最近一次 `market` 类型的 `analysis_runs`。

打开或刷新详情页只读取后台结果，不触发市场搜索或 AI 请求。

## 验证

- 已售物品不会被每日调度入队。
- 同一行情同一天重复调度或重试不会产生重复快照。
- 相同 `market_key` 的未售物品收到相同当日结果。
- 搜索失败、筛选失败及样本不足不会覆盖已有估值。
- 30 天、90 天、全部筛选以及涨跌、最高、最低计算正确。
- 零个、一个和多个快照均能正常渲染。
- 详情页打开和刷新不会调用估值接口。

## 完成标准

- 所有未售物品无需用户打开应用即可每日尝试刷新估值，成功结果写入快照，失败任务有可见状态。
- 资产详情能展示估值历史曲线，且今日参考市价仍是最突出的金额。
- 已售物品停止新增估值成本，但保留已有行情记录。
- 页面明确标注数据来自二手平台在售样本。
